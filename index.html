RL 算法工程师
📍 上海 / 北京 全职 / 实习
我们在找这样的你
  • 你不满足于只是调包 SFT，你相信 RL 才是模型获得真正推理能力的钥匙——让模型在试错中长出"思维"，而不是背诵答案
  • 你会盯着 wandb 的 reward curve 到凌晨，因为那个 spike 可能意味着某个 reward hacking，或者更妙的——某种 emergent reasoning 正在发生
  • 你读 GRPO / Reward Model / Verifier 的 paper 不是"跟风"，而是脑子里已经有好几个想魔改的点子，手痒想上集群跑
  • 你会和团队争论"这个 reward 到底该稀疏还是稠密"——然后直接写个 ablation 用数据说话
希望你具备
  • 把 PPO/GRPO 的 loss 手写在白板上不会卡壳，知道 advantage 估计的不同 trick 及其偏差-方差 tradeoff
  • 有至少一个 RL + LLM 的项目真正跑过——不是玩具实验，是真的在 GPU 集群上训过、调过、踩过坑的那种
  • PyTorch 是你的肌肉记忆，DeepSpeed/FSDP 至少摸过一种，能读懂 NCCL 的报错不靠 ChatGPT
  • 加分项：在 ICML/NeurIPS/ICLR 发过 RL 相关 paper；给 Open RLHF / verl 等开源项目提过 PR；自己 hack 过 reward model 的架构
LLM 后训练算法工程师
📍 上海 / 北京 全职 / 实习
我们在找这样的你
  • 你清楚 pre-training 给模型的是"知识"，而 post-training 给的是"灵魂"——你着迷于那层薄薄的对齐层如何彻底改变模型的可用性
  • 你对数据质量的洁癖到了偏执的程度——一条低质 SFT 样本混进十万条里你都能闻到，因为你知道 garbage in 在这里的代价是模型学会"假装听话"
  • 你不会满足于跑通一个 SFT 流程，你想搞清楚：什么样的数据配比能让模型不灾难性遗忘？curriculum 的顺序真的影响最终效果吗？合成数据和人工标注的最佳配比是多少？
  • RLHF/DPO/SimPO/KTO——你知道每个方法的数学本质、适用场景和隐藏坑点，并且在真实任务上对比过它们
希望你具备
  • 完整走过至少一轮 LLM 后训练全流程：数据清洗 → SFT → 对齐 → 评测，不是在 colab 上跑的 demo
  • 对数据有强迫症：会写复杂的 dedup / 质量过滤 / 格式校验 pipeline，知道 embedding-based dedup 和 exact match 分别适用什么场景
  • 熟悉主流评测体系（AlpacaEval / MT-Bench / Chatbot Arena 等），并且对它们的局限性有自己的判断——你不迷信 benchmark 数字
  • 加分项：自己做过数据合成（从 seed 生成到多轮过滤）；给开源模型做过完整的 post-training pipeline；在 alignment 方向有 paper 或技术博客

我们在找这样的你
  • 你相信代码不只是文本——它是可执行的逻辑，有结构、有依赖、有运行时反馈。你不想做又一个代码补全模型，你想做的是真正理解代码的 AI
  • 你对 SWE-bench 上的每个百分点都较真，但你更关心的是：模型是真的学会了推理 bug，还是只是在背 GitHub issue 里的讨论？
  • 你可能会在周末无聊时写个脚本，把所有开源 code LLM 在你的私有 repo 上跑一轮对比，然后得出某个反直觉的结论在 Twitter 上发个 thread
  • 你认为 code agent 的终极形态不是"写好 prompt 等输出"，而是让模型能 run test → read error → edit → run again——一个真正的编码闭环
希望你具备
  • 代码能力过硬：不止会写 Python，你对编译原理、AST、类型系统、程序分析至少一个方向有深入理解，能说出 tree-sitter 和 antlr 的设计差异
  • 有 code model 相关的 hands-on 经验：训练过、fine-tune 过、或者搭建过代码模型的评测 pipeline，知道 HumanEval 和 SWE-bench 的局限性
  • 理解 RL/搜索在代码生成中的应用：MCTS/过程奖励/执行反馈——你至少亲自实现过其中一种
  • 加分项：给开源 code model 项目贡献过代码；自己做过 code agent 或 coding benchmark；在编译器/PL/形式化验证方向有学术背景
AI Infra 训练推理框架工程师
📍 上海 / 北京 全职 / 实习
我们在找这样的你
  • 你对"快"有生理级别的追求——kernel launch overhead 多 5μs 你会难受，GPU 利用率掉到 60% 以下你会觉得在烧钱，memory bandwidth 是你每天盯的 dashboard
  • 你不只是"会用" vLLM 或 DeepSpeed——你读过它们的 CUDA kernel 源码，知道 page attention 为什么比 flash attention 在长序列上更吃显存，也改过 NCCL 的 ring/ tree topology 来适配你的集群拓扑
  • 别跟你提"用 huggingface trainer 一键启动"——你要的是能精准控制每一层的 activation checkpointing、能按 layer 粒度编排 offload、能在千卡上做到 55%+ MFU 的训练框架
  • 你享受那种"把一个 kernel 从 20ms 优化到 3ms"的纯粹快乐，然后轻描淡写地对同事说"就是把 shared memory 的 bank conflict 解决了一下"
希望你具备
  • CUDA 是你的第二语言：知道 warp divergence、shared memory bank conflict、occupancy 怎么调，能手写 flash attention 的前向 kernel
  • 分布式训练不只是概念：真正在百卡以上集群上调试过 TP/PP/DP/SP 的组合策略，遇到过一个 rank hang 住然后靠 trace 定位到某条链路故障的经历
  • 读过 Megatron-LM / DeepSpeed / FSDP 至少一个框架的核心代码，能说出它们的设计取舍——什么时候用 TP 而不是 PP？SP 的通信量和计算量怎么量化？
  • 加分项：给 vLLM / SGLang / TRT-LLM 提过 performance PR；搞过昇腾/寒武纪等国产芯片的算子适配；有 InfiniBand/RoCE 网络调优实战；写过 CUDA 的 flash-attn 风格 custom op

  • 你享受从 0 到 1 的构建感：周五下午聊了个 idea，周末撸出 MVP，周一早上的 demo 已经在处理真实用户反馈了

  • 后端：能用 Python（FastAPI）+ Go/TypeScript 搭建 agent 服务，处理过 SSE/WebSocket streaming，对并发和背压有自己的理解
  • Agent 基础扎实：Tool Use / Function Calling / ReAct / Plan-and-Execute 这些范式你都亲手实现过，能说出 LangChain 哪里设计得好、哪里有抽象泄漏

AI 产品经理
📍 上海 / 北京 全职 / 实习
我们在找这样的你
  • 你不会在 PRD 里写"接入大模型实现智能问答"就觉得万事大吉——你会追问：模型的 latency 目标是多少？如果幻觉率 5% 用户能接受吗？failure mode 是什么样的？fallback 策略是什么？因为你知道 AI 产品的难度不在"能跑"，而在"永远不崩"
  • 你每天都用各种 AI 产品——从 ChatGPT 到 Cursor 到 Perplexity 到各种小众 agent tool——你不是"调研"，你是真的重度用户，你会因为某个交互细节不爽而在群里喷三屏
  • 你能和算法同学在同一个频道上讨论问题：知道 SFT/RLHF/RAG/Agent 分别能解决什么、不能解决什么，不会被"这个模型 benchmark 很高"这种话糊弄——你会反问"什么 benchmark？和我们的用户场景有什么关系？"
  • 你相信最好的 AI 产品经理是半个 prompt engineer + 半个 UX 设计师 + 半个数据分析师，并且你已经在这三个方向上都有扎实的积累
希望你具备
  • 有实际交付过的 AI 产品案例——不一定是公司项目，可以是个人 side project、开源工具、甚至一个活跃的 AI 社区 bot，但必须是你独立负责产品方向和关键决策
  • 技术理解力：能自己调 API 跑通一个 agent demo，能看懂模型 benchmark 的论文摘要，能和工程师讨论 embedding/token/context window 这些概念而不需要对方反复解释
  • 产品基本功：能从用户反馈中提炼出真正的需求（而不是用户说什么就做什么），能用数据验证假设，能在一周内把一个模糊的 idea 变成一个可评审的产品方案
  • 加分项：有开发者工具/coding 工具/agent 平台类产品的经验；自己写过代码或做过数据分析；在 AI 社区有影响力（Twitter/即刻/公众号/播客等）
