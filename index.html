RL 算法工程师
📍 上海 / 北京 全职 / 实习
你将挑战的问题
  • 设计可验证奖励体系（RLVR），让模型在数学、代码、工具调用等有"标准答案"的任务上自我进化，而不是靠人类偏好硬喂
  • 攻克长链条推理中的信用分配（credit assignment）难题：一条几千 token 的推理轨迹，哪一步该被奖励、哪一步该被惩罚
  • 把 PPO / GRPO / DPO / RLOO 玩透，不止是调参侠，而是能讲清楚每一种目标函数背后的偏差与方差权衡
  • 探索 Agentic RL：多轮工具调用、多智能体协作场景下的奖励设计与训练稳定性
我们希望你
  • 扎实的强化学习基础，复现过至少一个顶会 RL / LLM-RL 工作的核心结果，而不是停留在读论文
  • 熟悉大规模 RLHF/RLAIF 训练 pipeline，踩过 reward hacking、KL 爆炸、训练崩溃的坑
  • 对"模型如何钻奖励的空子"有本能的警觉，写 reward function 之前先扮演一次"最坏情况的模型"
  • 加分项：发表过 RL / LLM 相关顶会论文，或有开源 RL 训练框架贡献经历
LLM 后训练算法工程师
📍 上海 / 北京 全职 / 实习
你将挑战的问题
  • 构建 SFT → RM → RL 的完整后训练链路，让每一轮迭代都有可衡量的能力提升，而不是"感觉变好了"
  • 设计数据飞轮：模型生成 → 自动/人工评估 → 高质量数据回流 → 再训练，让数据质量随迭代自增强
  • 解决对齐税（alignment tax）问题：如何在提升安全性/可控性的同时不牺牲模型的推理与创造力上限
  • 探索测试时计算（test-time compute）与后训练的结合：如何让模型"学会"更好地利用推理时的算力
我们希望你
  • 对 SFT、DPO、RLHF 全链路有一手实践经验，知道每个阶段该看什么指标、该在哪里止损
  • 具备扎实的数据敏感度，能设计出有区分度的评测集，而不是被 benchmark 分数忽悠
  • 熟悉主流开源模型的后训练技术报告，能快速判断一个技巧是否值得复现
  • 加分项：主导过千卡以上规模的训练任务，或有 reward model / evaluation 框架的设计经验
Code 算法工程师
📍 上海 / 北京 全职 / 实习
你将挑战的问题
  • 让模型从"写代码"进化到"写对代码"：引入可执行反馈（execution feedback）、单元测试自动生成与自我验证
  • 攻克 repo 级别的代码理解与生成：跨文件依赖、长上下文代码库导航、真实工程场景下的多步修改
  • 设计 SWE-bench 式的能力评测与训练闭环，让模型具备"提 PR"级别的软件工程能力
  • 探索 Code Agent 与 RL 的结合：用编译器/测试结果作为可验证奖励信号驱动训练
我们希望你
  • 有扎实的软件工程功底，而不只是"会训模型"——你需要理解真实代码库长什么样
  • 熟悉代码大模型的训练与评测体系（HumanEval、SWE-bench 等），知道这些 benchmark 的局限在哪
  • 具备构建自动化评测/沙盒执行环境的工程能力
  • 加分项：有开源项目高质量贡献记录，或参与过代码大模型的预训练/后训练

  • 加分项：有千卡以上集群训练调优经验，或在开源训练/推理框架有 merged PR
